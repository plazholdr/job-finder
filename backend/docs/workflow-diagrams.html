üéì Diagram 1: Student Registration Workflow

üîÑ Flow Steps:

1. User opens the WEB website
2. Decision: User login?
3. If NO: User finds "Username/Email can be email or Password"
4. User gets redirected back to the WEB system homepage after successful verification
5. User input their profile information (First name, Last name, Email, Phone, Bio, IC/Passport no, not email)
6. User input their education background (Level of education, Institution name, Qualification, Start date, End date)
7. User input their certification (multiple) - Certificate title, Certificate issuer, Acquired date, Description
8. User input their interest (multiple) - Interest title, Interest description, Since
9. User input their previous work experience (multiple) - Event name, Event position, Event date (start-end), Event description
10. User information will be stored to the db
11. User gets redirected back to the WEB system homepage

üè¢ Diagram 2: Company Registration Workflow

üîÑ Flow Steps:

1. Company admin opens the WEB website
2. Decision: Company login?
3. If NO: Company admin inputs Username/Email and Password
4. Decision: Account exists?
5. If NO: Company admin inputs personal information (First name, Last name, Email, Phone)
6. Company admin inputs company information (Company name, Company registration number, Company industry, Company size, Company website, Company description, Company business nature, Company address)
7. Decision: Upload company documents?
8. If YES: Company admin uploads documents (Company logo, Company registration certificate, Company business license, etc.)
9. Company admin views the verification status
10. Company admin views company profile
11. Company admin can perform the functions below (Search for interns, Filter list of interns, Shortlist interns, View intern profile, Create intern profile, Invite intern for interview)

üåê API Endpoints:

Company Registration:

POST /users Content-Type: application/json

Company Profile Creation:

POST /companies Authorization: Bearer <accessToken> Content-Type: application/json

Document Upload:

POST /upload Authorization: Bearer <accessToken> Content-Type: multipart/form-data

Company Verification:

POST /company-verifications Authorization: Bearer <accessToken> Content-Type: application/json

üìã Diagram 3: Intern Profile Update Workflow

üîÑ Flow Steps:

1. Intern clicks on the profile button
2. Intern sees the existing profile information
3. Intern clicks on the update profile button
4. Decision: Update profile OR privacy setting?
5a. If profile update: Intern inputs the new value
5b. If privacy setting: Intern chooses the new privacy setting
6. The new information/privacy setting will be updated

üåê API Endpoint:

PATCH /users/:id Authorization: Bearer <accessToken> Content-Type: application/json

üîí Privacy Setting Options:

1. "full" (Default): Full access - can see everything except phone number

3. "private": Cannot be searched/displayed when search through URL directly
